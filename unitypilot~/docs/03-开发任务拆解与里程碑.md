# 03-开发任务拆解与里程碑

## 1. 目标
将 `UnityPilot MCP` 从文档阶段推进到可运行 MVP，覆盖：
1. Unity 连接与协议通信。
2. 编译与错误回传。
3. 自动修复循环。
4. PlayMode 与鼠标事件模拟。

## 2. 里程碑总览
| 里程碑 | 周期 | 目标 | 验收输出 |
| --- | --- | --- | --- |
| Milestone A | 第 1 周 | 打通 WS 基础链路 | Unity 可连接、hello/heartbeat 正常 |
| Milestone B | 第 2 周 | 打通编译与错误收集 | compile + compile_errors 完整可用 |
| Milestone C | 第 3 周 | 打通自动修复闭环 | 至少 1 个真实错误场景自动修复成功 |
| Milestone D | 第 4 周 | 打通 PlayMode 与鼠标模拟 | play/stop + click/drag/up/down 可执行 |

## 3. 模块任务拆解
## 3.1 M01 编排服务模块（Python）
### 任务
1. 建立 WebSocket Server（`127.0.0.1:8765`）。
2. 实现会话注册、心跳、断线清理。
3. 实现命令分发器与超时控制。
4. 暴露 MCP 工具函数（先做 compile/playmode/mouse）。

### 完成标准
- 可同时处理命令请求与事件回包。
- 命令超时返回标准错误对象。
- 所有请求带 `requestId` 且可追踪。

## 3.2 M02 UnityBridge通信模块（Unity Editor）
### 任务
1. Editor 启动自动连接 Orchestrator。
2. 实现 `session.hello`、`heartbeat`、命令路由。
3. 统一回包 `result/error/event`。
4. 断线自动重连。

### 完成标准
- Unity 启动后 10 秒内可连上。
- 断线后可自动重连。
- 未注册命令返回 `COMMAND_NOT_FOUND`。

## 3.3 M03 编译与错误收集模块
### 任务
1. 实现 `compile.request`。
2. 实现 `compile.status` 事件上报。
3. 实现结构化错误输出（file/line/column/message/severity）。
4. 防重复编译（busy）。

### 完成标准
- 可稳定触发编译并回传状态。
- 能拿到可用于自动修复的结构化错误。
- 编译中重复触发返回 `EDITOR_BUSY`。

## 3.4 M06 代码补丁执行模块
### 任务
1. 实现补丁应用器（replace_block / replace_all / overwrite）。
2. 增加写入前校验与写入后校验。
3. 增加原子写入与失败回滚。
4. 输出差异摘要。

### 完成标准
- 写入失败不污染源文件。
- 唯一匹配失败时明确报错。
- 每轮有补丁执行报告。

## 3.5 M04 自动修复循环模块
### 任务
1. 实现固定状态机：`compile -> errors -> patch -> compile`。
2. 支持 `maxIterations` 上限。
3. 支持 `auto_fix_stop`。
4. 输出每轮状态与最终报告。

### 完成标准
- 无错误时自动结束。
- 到达上限后正确退出。
- 可中途停止且状态正确。

## 3.6 M05 PlayMode与输入模拟模块
### 任务
1. 实现 `playmode.set(play/stop)`。
2. 实现 `mouse.event`（left/middle/right + down/move/drag/up/click）。
3. 加入窗口聚焦与坐标映射。
4. 异常补偿 `MouseUp`。

### 完成标准
- 3 秒内可进入/退出 PlayMode。
- Scene/Game 窗口可执行鼠标动作并返回成功结果。

## 5. 联调计划
## 5.1 联调顺序
1. M01 + M02：握手、心跳、基础命令。
2. 接入 M03：编译状态与错误。
3. 接入 M06 + M04：自动修复闭环。
4. 接入 M05：PlayMode 与鼠标输入。

## 5.2 每日联调检查
1. WS 连接是否稳定（掉线次数、重连次数）。
2. 命令成功率（成功/失败/超时）。
3. 编译闭环成功率（从有错到无错）。
4. 鼠标事件成功率（按窗口统计）。

## 6. 风险与应对
| 风险 | 影响 | 应对 |
| --- | --- | --- |
| Unity 编辑器事件时序差异 | 命令偶发失败 | 增加状态确认与重试策略 |
| 编译错误解析不稳定 | 自动修复失败率高 | 固化错误结构字段并做兜底解析 |
| 鼠标注入窗口焦点异常 | 无法复现操作 | 统一先聚焦再注入 |
| 自动修复循环死循环 | 资源浪费 | 强制 `maxIterations` + 可停止命令 |

## 7. 验收门槛（MVP）
1. 连通性：连续运行 30 分钟，连接不断或可自动恢复。
2. 编译能力：至少 3 个典型错误样例可被识别并结构化输出。
3. 自动修复：至少 1 个真实场景从编译失败到通过。
4. 交互能力：PlayMode 与鼠标事件在 Windows/macOS 均通过基本用例。
