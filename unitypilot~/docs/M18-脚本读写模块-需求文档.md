# M18-脚本读写模块-需求文档

## 1. 模块职责
- 负责：
  - 通过 Unity AssetDatabase API 安全读取、创建、更新、删除 C# 脚本文件。
  - 确保文件操作后自动触发 Unity 资源刷新（Refresh）与重导入。
  - 处理脚本编码的一致性（UTF-8）与行尾序列转换。
- 不负责：
  - 不负责脚本内容的语法分析或代码生成逻辑。
  - 不负责跨文件的重构操作（如全局重命名类名）。
  - 不负责对非脚本资产（如贴图、材质）的二进制操作。
- 输入/输出：
  - 输入：`script.read`, `script.create`, `script.update`, `script.delete` 命令及路径参数。
  - 输出：文件内容字符串、操作成功状态、AssetDatabase 错误信息。

## 2. 数据模型
| 字段名 | 类型 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- |
| scriptPath | string | 脚本在 Assets 下的相对路径 | 以 Assets/ 开头且后缀为 .cs | "" |
| content | string | C# 源码正文 | 有效的文本字符串 | "" |
| encoding | string | 文件字符编码 | "utf-8" | "utf-8" |
| lastModified | long | 最后修改时间戳（毫秒） | > 0 | 0 |
| guid | string | Unity 资源的唯一标识符 | 32 位十六进制字符串 | "" |

## 3. CRUD 操作
| 操作 | 入口 | 禁用条件 | Command类名 | Undo语义 |
| --- | --- | --- | --- | --- |
| 读取脚本 | script.read | 文件不存在 | ReadScriptCmd | 不进 CommandHistory |
| 创建脚本 | script.create | 路径已存在或父目录缺失 | CreateScriptCmd | 不进 CommandHistory |
| 更新脚本 | script.update | 文件被外部锁定 | UpdateScriptCmd | 不进 CommandHistory |
| 删除脚本 | script.delete | 文件被版本控制锁定 | DeleteScriptCmd | 不进 CommandHistory |
| 同步资源 | script.refresh | 编译中 | RefreshAssetCmd | 不进 CommandHistory |

## 4. 交互规格
- 触发事件：MCP 发起 script.* 系列请求。
- 状态变化：Unity 侧文件系统写入 -> AssetDatabase.ImportAsset -> 触发编译。
- 数据提交时机：文件写入磁盘且 Unity 确认导入成功后立即返回。
- 取消/回退：操作具有原子性，失败时会自动抛出异常；不支持由本模块管理的逻辑撤销。

## 5. 视觉规格
不涉及。

## 6. 校验规则
| 规则 | 检查时机 | 级别 | 提示文案 |
| --- | --- | --- | --- |
| 路径合法性校验 | 任何操作前 | Error | 路径必须位于 Assets 目录下且以 .cs 结尾 |
| 内容长度限制 | 写入前 | Warning | 脚本内容超过 1MB，可能导致编辑器卡顿 |
| 文件存在性冲突 | 创建前 | Error | 目标位置已存在同名文件，请使用 update |
| 编译锁检测 | 操作前 | Warning | 当前正在编译，资源导入可能存在延迟 |

## 7. 跨模块联动
| 模块 | 方向 | 说明 |
| --- | --- | --- |
| M02 UnityBridge通信 | 被动接收 | 承载 WebSocket 请求与响应的序列化传输 |
| M03 编译与错误收集 | 主动通知 | 脚本变更后自动触发该模块的编译状态监听 |
| M12 Asset资产管理 | 被动接收 | 共享 AssetDatabase 相关的锁定与同步逻辑 |

## 8. 技术实现要点
- 关键类与职责：
  - ScriptFileHandler：负责底层 System.IO 与文本编码处理。
  - UnityAssetAdapter：封装 AssetDatabase API 调用。
  - ChangeNotificationRelay：负责向 Unity 编辑器发送 ImportAsset 通知。
- 核心流程：receive command -> validate path -> disk io -> AssetDatabase.Refresh -> await import completion -> response。
- 性能约束：
  - 100KB 以下文件读取耗时 < 50ms。
  - 文件变更后的 ImportAsset 触发延迟 < 100ms。
  - 内存中不持久化存储大文件的全文副本。

## 9. 验收标准
1. [空路径] -> [调用 script.create 写入有效代码] -> [Assets 目录生成 .cs 文件且 Unity 自动选中]。
2. [已有脚本] -> [调用 script.read] -> [返回包含 line endings 规范化的完整源码]。
3. [已有脚本] -> [调用 script.update 修改类名] -> [文件内容更新且 Unity 控制台无重导入错误]。
4. [非法路径] -> [执行 script.delete] -> [返回 400 路径校验失败错误]。

## 10. 边界规范
- 空数据：写入空字符串内容时，生成仅包含注释的占位文件。
- 单元素：支持对深度嵌套目录下的单个文件进行读写。
- 上下限临界值：支持处理包含 50,000 行代码的超大型脚本文件。
- 异常数据恢复：磁盘写入失败时，AssetDatabase 不会执行 Refresh 以防止损坏旧资源引用。
