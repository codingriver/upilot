# 可选与扩展能力 — 需求清单（含详细开发需求）

本文将 `02-MCP`、`01-WS`、M26、M27 正式需求中出现的**可选能力**与**可选字段**整理为可排期清单；每项包含**来源、含义、详细开发需求、验收要点**。

### 建议优先级总览（供排期勾选）

说明：**P0** = 契约/安全/误用风险，应尽快对齐；**P1** = 核心体验或 E2E 可靠性，建议本迭代或下一迭代；**P2** = 增强与文档/运维便利；**P3** = 远期、按需或大投入。**优先级为建议值**，你可按业务直接改档。

| 编号 | 建议优先级 | 一句话理由 |
|------|------------|------------|
| **A-01** `open_editor` 的 `command` 可选 | **P1** | 手开 Unity + 仅等待连接是常见工作流，缺了要反复绕路。 |
| **A-02** `compileRequestId` 可选 | **P2** | 省参数、改善 DX；有明确 `compileRequestId` 时也可不用。 |
| **A-03** `unity.editor_state` P1 / 契约稳定 | **P2** | 多为「已实现则验收 + 文档纠偏」；新开发价值次于交互类。 |
| **A-04** `scrollViewNamePath` 可选 | **P1** | 嵌套 ScrollView 场景下否则无法稳定操作；与 M27 强相关。 |
| **A-05** `rshell.status` 空 `connectionId` 列全部 | **P2** | 多会话运维/排障有用，非主路径阻塞。 |
| **B-01** G1 UTF 读规格 | **P3** | 双栈维护成本高，除非强需求「无 MCP CI」。 |
| **B-02** 非目标「不替代 UTF」 | **—** | 文档/原则，不是可开发功能项。 |
| **B-03** G3 视觉断言截图可选 | **P1** | 无头/弱 GPU 环境可跑通主流程。 |
| **B-04** Phase B（UTF 对齐 / DriverTests / M17） | **P3** | 与 B-01 同类，投入大、收益看团队 CI 策略。 |
| **B-05** Phase C（帧级 wait、console snapshot 等） | **P3** | 有明确竞态缺口再开，避免过早抽象。 |
| **B-06** `reflection.get` 断言 | **P2** | 补覆盖快，依赖 M20 边界清晰即可。 |
| **B-07** `assert.screenshot` + `optional: true` | **P1** | 与 B-03/B-11 配套，报告语义要一致。 |
| **B-08** `teardown` 可选清理 | **P1** | 长套件稳定性、资源回收。 |
| **B-09** `artifact_dir` 可选默认 | **P2** | 改善开箱体验；手动传路径也能用。 |
| **B-10** `until: editor_idle` 等扩展 | **P3** | 先有 `wait(ms)` + 编译等待通常够用。 |
| **B-11** 截图部分平台 `skipped` | **P1** | 避免误报失败、与风险表一致。 |
| **C-01** ZIP / Webhook / 云端可选 | **P2** | 集成与运维增强；核心 E2E 不依赖。 |
| **C-02～C-03** `matchMode` / `scrollbarAxis` / `dragSteps` 等可选字段 | **P1** | M27 正式字段表；未实现则与文档不一致，建议与 A-04 同批验收。 |

**若只做一版「最小可交付」组合（示例）**：A-01、A-04、B-03、B-07、B-08、B-11、C-02～C-03（其余按表顺延或砍到 P3）。

## A. MCP / WS 通用与工具层（02 / 01）

### A-01 `unity.open_editor` — `command` 可选

| 项目 | 内容 |
|------|------|
| **文档位置** | `docs/02-MCP工具接口定义.md`（`unity.open_editor` / `open_editor`） |
| **含义** | 启动 Unity 的可执行路径或命令**可为空**：为空时不主动拉起进程，仅**等待**已有 Unity 通过 Bridge 连接 MCP。 |
| **详细开发需求** | 1. Orchestrator 解析 `command`：空字符串 / 未传 → **不**调用进程启动逻辑，仅进入 `waitForConnect` 循环。<br>2. 非空时按平台启动 Unity 并继续等待同一 `sessionId`/WS 握手。<br>3. 错误码：`UNITY_NOT_CONNECTED` 时在响应 `detail` 中区分「从未连接」与「超时」。<br>4. 文档与 MCP 工具参数说明保持一致（`command` 默认 `""` 或未暴露必填）。 |
| **验收要点** | `command` 省略时，在 Unity 已手动打开并连接的情况下，返回 `connected=true`；未打开时在 `waitForConnectMs` 内超时返回失败。 |

---

### A-02 `compileRequestId` — 请求体可选

| 项目 | 内容 |
|------|------|
| **文档位置** | `docs/02-MCP工具接口定义.md` — `compile_status`、`compile_errors` 等 |
| **含义** | 查询编译状态/错误时，可不携带最近一次编译的 `compileRequestId`，由 Bridge **推断当前会话默认编译任务**或返回「无活跃 compileRequest」。 |
| **详细开发需求** | 1. Python `tool_facade` 与 Bridge 载荷：`compileRequestId` 为可选字段；缺省时 Bridge 行为需**唯一确定**（例如：返回最近一次 `compile.request` 的 id，或明确返回 `idle` + 提示无 id）。<br>2. `02` 与 `01-WS` 中同一命令的 payload 字段一致。<br>3. 若存在多并发编译请求，文档需说明**缺省 id 时绑定哪一条**（通常「最后一次成功接受的 request」）。 |
| **验收要点** | 不传 `compileRequestId` 时接口不 500；行为与文档「默认指向」一致。 |

---

### A-03 `unity.editor_state` — P1 / 非 MVP 标注

| 项目 | 内容 |
|------|------|
| **文档位置** | `docs/02-MCP工具接口定义.md` 工具表、`§4.10` |
| **含义** | 工具在能力矩阵中标为**非 MVP**，不阻塞首版发布，但**仍应实现并保持契约稳定**。 |
| **详细开发需求** | 1. `editor.state` WS 命令与 `unity_editor_state` MCP 工具已存在时：保证返回字段与 `§4.10` 示例一致（`connected`、`isCompiling`、`playModeState`、`activeScene` 等）。<br>2. 若总览仍写「P1 新增」，需在 `00` 或本清单中**标注已实现**，避免「文档写未做、代码已做」。<br>3. 可选扩展：与 `resource.editorState` 去重策略在 `M23` 说明中一致。 |
| **验收要点** | 连接 Unity 后调用 `unity_editor_state` 返回 `ok=true` 且字段类型稳定。 |

### A-04 `uitoolkit.scroll` / `uitoolkit.scrollbar.drag` — `scrollViewNamePath` 可选

| 项目 | 内容 |
|------|------|
| **文档位置** | `docs/01-WS协议定义.md`、`docs/02-MCP工具接口定义.md` §4.11 |
| **含义** | 嵌套 `ScrollView` 时可用 **`scrollViewNamePath`**（`outer|inner` 或 `/` 分隔）；**不传**时保持原语义：单 `elementName` / `elementIndex` / 第一个 `ScrollView`。 |
| **详细开发需求** | 1. **Bridge**：`UIToolkitScrollPayload`、`UIToolkitScrollbarDragPayload` 含 `scrollViewNamePath`；`ResolveScrollViewByPath` 自根向内逐层 `Q<ScrollView>(name)`。<br>2. **Python**：`tool_facade` / `mcp_stdio_server` 暴露可选参数；`editor_e2e` `actions.py` 透传 YAML 字段。<br>3. **错误**：路径无效时 `SCROLLVIEW_NOT_FOUND` 或统一 `state` 文案。<br>4. **文档**：与 M27 BL-07「仅文档 vs 名称链」表述一致，避免与「depth」旧表述冲突（以 `scrollViewNamePath` 为准）。 |
| **验收要点** | 不传路径时行为与改路径前一致；传合法双层路径时滚动/拖拽作用于内层。 |

---

### A-05 `rshell.status` — `connectionId` 可选（空 = 列出全部）

| 项目 | 内容 |
|------|------|
| **文档位置** | `docs/01-WS协议定义.md` §5.6、`M25` |
| **含义** | `connectionId` **为空或未传**时返回**当前所有 RShell 会话摘要**；传入则返回**单连接**详情。 |
| **详细开发需求** | 1. **Bridge**：`RShellStatusPayload.connectionId` 可选；空时构造 `RShellStatusResultPayload.connections[]`。<br>2. **Python**：`rshell_status(connection_id="")` 与 MCP `unity_rshell_status(connectionId="")` 一致。<br>3. **安全**：列表不得输出密钥；仅 host/port/status/error/lastHeartbeat。 |
| **验收要点** | 无连接时返回空数组或明确 `ok` + 空列表；多连接时条数与 `Sessions` 一致。 |

## B. M26 — 编辑器 E2E 自动化测试（开发方案中的可选项）

**总文档**：`docs/M26-编辑器E2E自动化测试-开发方案.md`

### B-01 G1 — 规格可被（可选）UTF 读取

| 项目 | 内容 |
|------|------|
| **含义** | 同一份 YAML **可**被 Unity Test Framework **在无 MCP 环境**下读取并执行等价断言（**可选**，非必须）。 |
| **详细开发需求** | 1. 定义**最小 YAML 子集**与 UTF 可执行的映射规则（或放弃并文档标注「不实现 UTF 读 YAML」）。<br>2. 若实现：在 `unitypilot-editor/Tests/Editor/` 提供 **loader** 或 **代码生成**（YAML → C# 测试桩）。<br>3. 与 **Phase B** 共用接口（见 B-05）。 |
| **验收要点** | 文档与代码二选一明确：**已实现 UTF 路径**或**明确声明不实现**。 |

---

### B-02 非目标 — 不替代 UTF

| 项目 | 内容 |
|------|------|
| **含义** | E2E 编排**不替代** Unity Test Framework 的单元/集成测试定位。 |
| **详细开发需求** | 1. 文档与培训材料中保持表述：**M26 = 编排 + 编辑器侧验收**；**M17 `test.run`** = UTF。<br>2. 不在 `editor_e2e` 中重复实现 NUnit 断言逻辑（避免双维护）。 |
| **验收要点** | 架构说明可追溯，无「E2E 取代单元测试」歧义。 |

---

### B-03 G3 — 视觉断言「截图可选」

| 项目 | 内容 |
|------|------|
| **含义** | 断言可以**不依赖**截图；截图仅作**辅助附件**或可选断言。 |
| **详细开发需求** | 1. `assert` 默认以 `uitoolkit.query` / `console` 等为主；`screenshot` 类断言支持 `optional: true`。<br>2. 失败报告：截图失败时**不掩盖**主断言失败原因（见 M26 风险表）。 |
| **验收要点** | 无 GPU/无窗口环境仍能通过纯 query/console 用例。 |

### B-04 Phase B（可选）— UTF 对齐、`IEditorE2EAssertions`、`EditorE2EDriverTests`、与 M17 联调

| 项目 | 内容 |
|------|------|
| **含义** | 在 **EditMode** 跑与 E2E **等价**的断言，便于 CI **无 MCP**。 |
| **详细开发需求** | 1. 定义接口 `IEditorE2EAssertions`（或等价抽象），含 query/dump/console 比较签名。<br>2. `EditorE2EDriverTests.cs`：至少 **1～N 条**与 `e2e-specs/examples` 对齐的 smoke。<br>3. CI：`unity_test_run` 能收集到该程序集（程序集定义、`*.asmdef` 引用 Editor）。<br>4. 与 M17 `test.run` 参数（`testFilter`）可筛选 `EditorE2EDriverTests`。 |
| **验收要点** | 本地执行 `unity_test_run(EditMode)` 可发现并通过（或显式 Skip）上述用例。 |

### B-05 Phase C（按需）— `editor.wait` 帧级、`console.snapshot` token 等 primitive

| 项目 | 内容 |
|------|------|
| **含义** | 仅当 Phase A 出现**明确竞态/日志增量**缺口时，在 Bridge **增量**增加通用命令。 |
| **详细开发需求** | 1. **帧级等待**：`editor.wait` 扩展 `mode=frames` + `frameCount`（需主线程排队实现）。<br>2. **console.snapshot**：执行 `console.read` 前后各一次，返回 `snapshotId`，断言步骤只比对**增量**（需 M07 支持或 Bridge 缓存）。<br>3. 每增一项：**更新 `01-WS`、`02-MCP`、OperationTracker、dispatcher 超时**。 |
| **验收要点** | 有独立 issue/里程碑触发再开发；未开发前文档保持「按需」。 |

### B-06 断言 `reflection.get`（可选 Phase B）

| 项目 | 内容 |
|------|------|
| **含义** | E2E 规格中可对**静态/反射路径**读值并断言（依赖 M20）。 |
| **详细开发需求** | 1. `editor_e2e` 的 `assert.type: reflection.get` 与 `actions.py` 映射到 `reflection.*` 或 `csharp.execute`（需安全边界）。<br>2. 与 Phase B UTF 共用同一「期望值」比较逻辑（若做 B-04）。 |

### B-07 `assert.screenshot` + `optional: true`

| 项目 | 内容 |
|------|------|
| **含义** | 截图断言失败时，若 `optional: true`，**不导致整步失败**（仅存档）。 |
| **详细开发需求** | 1. `assertions.py` 解析 `optional`；为真时捕获截图错误并写入 `detail.warning`。<br>2. 报告 `passed` 逻辑：主断言仍失败则整步失败；仅截图失败且 optional 时步骤可标 `ok` + warning。 |
| **验收要点** | 单元测试或手工：optional 截图失败时 `passed` 与预期一致。 |

### B-08 `teardown` — 可选清理

| 项目 | 内容 |
|------|------|
| **含义** | 规格中 `teardown` **可省略**；非空时在 `steps` 之后执行，用于关窗、还原菜单等。 |
| **详细开发需求** | 1. `runner.py` 在 `steps` 结束后**无论成败**尽量执行 `teardown`（或文档约定仅成功时执行 — 需统一）。<br>2. `teardown` 与 `setup` 共用同一 `action` 枚举与 `merge_payload`。 |
| **验收要点** | 省略 `teardown` 的规格可运行；含 `teardown` 时在报告中单独阶段或合并 step 索引。 |

### B-09 `artifact_dir` — 可选参数

| 项目 | 内容 |
|------|------|
| **含义** | `unity_editor_e2e_run` 可不指定产物目录，使用**默认临时目录**或仓库约定路径。 |
| **详细开发需求** | 1. `tool_facade` / MCP：`artifactDir` 默认 `None` → runner 内 `tempfile.mkdtemp` 或 `.unitypilot/e2e-artifacts/<runId>`。<br>2. 文档写明默认路径规则与磁盘清理策略（是否保留失败用例）。 |
| **验收要点** | 不传 `artifactDir` 仍能生成 `report.json`。 |

### B-10 `wait` / `until: editor_idle` — 文档扩展可能

| 项目 | 内容 |
|------|------|
| **含义** | 当前以 `ms` + `editor.delay` 为主；**未来**可支持 `until: editor_idle` 等条件等待。 |
| **详细开发需求** | 1. **未实现前**：规格中仅支持 `wait` + `ms`（及 `unity` 开关）；YAML 中若出现 `until` 应**校验报错**或**显式忽略并记 warning**。<br>2. **实现时**：需 Bridge 提供「编译空闲/PlayMode 静止」等查询或与现有 `compile_wait` 组合。 |
| **验收要点** | 文档与 `actions.py` 支持的字段一致，避免静默忽略。 |

### B-11 风险表 — `screenshot` 部分平台、`skipped`

| 项目 | 内容 |
|------|------|
| **含义** | 截图在部分环境不可用时应 **`skipped`**，且**不作为**唯一门禁。 |

| **验收要点** | 模拟无窗口标题时报告与验收文档一致。 |

### C-02 字段表 — 可选参数（开发约束）

以下字段在协议与 MCP 中均为**可选**，缺省时行为以需求文档 **默认值** 为准；实现须 **Bridge + Python + E2E 映射** 全链路一致。

| 字段 | 适用命令/场景 | 默认值（文档） | 详细开发需求 |
|------|----------------|------------------|--------------|
| **matchMode** | `editor.window.close` / `setRect` | `exact` | 枚举 `exact` \| `contains`；`contains` 多匹配时文档约定「取首个」+ 可选 `multipleMatches` 输出。 |
| **scrollbarAxis** | `uitoolkit.scrollbar.drag` | `vertical` | `horizontal` \| `vertical`；选择对应 `Scroller`。 |
| **dragSteps** | `uitoolkit.scrollbar.drag` | `5` | 范围建议 `[2,20]`；C# 校验越界时钳制或返回 `INVALID_PAYLOAD`。 |
| **wheelDeltaX / wheelDeltaY** | `uitoolkit.event` + `wheel` | `0` | IMGUI `Event.delta`；与 RShell 无关。 |
| **exportZip** | `unity_editor_e2e_run` | `false` | 为 `true` 时打包 `artifactDir` 下已有文件为 `e2e-bundle.zip`，响应含 `zipPath`。 |
| **webhookOnFailure** | `unity_editor_e2e_run` | `false` | 仅失败且 `UNITYPILOT_E2E_WEBHOOK_URL` 有效时 POST。 |

**补充（同文档字段表）**：`scrollViewNamePath`、`wheelDelta*`、`closable` 等见 **A-04** 与 M27 §6 错误码表。

### C-03 `matchMode`、`scrollbarAxis`、`dragSteps` — 专项说明

| 字段 | 详细开发需求（补充） |
|------|----------------------|
| **matchMode** | Bridge 窗口查找：先 trim，非法值回退 `exact`；`contains` 时大小写策略与文档一致（通常 OrdinalIgnoreCase）。 |
| **scrollbarAxis** | 拖拽前校验目标 `ScrollView` 是否存在对应轴 `Scroller`；无则 `SCROLLER_NOT_FOUND`。 |
| **dragSteps** | 插值点含起点/终点；鼠标事件坐标在 `Scroller` 可视区域内；与 `normalizedThumbPosition` 联立时的几何在 `UnityPilotUIToolkitService` 内注释说明。 |
