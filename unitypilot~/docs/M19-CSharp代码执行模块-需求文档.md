# M19-CSharp代码执行模块-需求文档

### 1. 模块职责

- 负责接收外部输入的 C# 代码片段并在 Unity 编辑器进程中通过 Roslyn 脚本引擎实时编译执行。
- 负责管理代码执行状态，包括排队、运行中、成功完成、失败报错及超时处理。
- 负责捕获并返回脚本执行的序列化结果或编译/运行期异常信息。
- 不负责持久化存储执行的代码脚本。
- 不负责处理涉及 Unity 编辑器 UI 阻塞交互的代码。
- 不负责多线程环境下的代码同步竞态保护。

### 2. 数据模型


| 字段名             | 类型     | 语义         | 合法范围                                     | 默认值     |
| --------------- | ------ | ---------- | ---------------------------------------- | ------- |
| executionId     | string | 执行任务唯一标识   | UUID 格式                                  | 无       |
| code            | string | 待执行的 C# 源码 | 长度 1-50000 字符                            | 无       |
| status          | string | 执行状态       | pending/running/completed/failed/timeout | pending |
| result          | string | 脚本返回值序列化文本 | 任意字符串                                    | 空       |
| error           | string | 错误或异常详细描述  | 任意字符串                                    | 空       |
| executionTimeMs | long   | 实际执行耗时（毫秒） | >= 0                                     | 0       |
| startedAt       | long   | 开始执行的时间戳   | Unix 时间戳                                 | 0       |


### 3. CRUD 操作


| 操作   | 入口             | 禁用条件          | Command 类名            | Undo 语义           |
| ---- | -------------- | ------------- | --------------------- | ----------------- |
| 执行代码 | csharp.execute | 编译中或脚本包含禁用关键字 | ExecuteCSharpCmd      | 不进 CommandHistory |
| 查询状态 | csharp.status  | 无             | GetExecutionStatusCmd | 不进 CommandHistory |
| 终止执行 | csharp.abort   | 状态为非运行中       | AbortExecutionCmd     | 不进 CommandHistory |


### 4. 交互规格

- **触发事件**：客户端通过 WebSocket 发送 `csharp.execute` 消息触发。
- **状态变化**：接收后立即返回 `pending`，进入队列后转为 `running`，结束后转为终态。
- **数据提交时机**：代码执行完毕并完成结果序列化后立即推送状态更新。
- **取消回退**：执行过程不支持业务级回退，仅能强制终止线程。

### 5. 视觉规格

不涉及。

### 6. 校验规则


| 规则      | 检查时机 | 级别      | 提示文案                            |
| ------- | ---- | ------- | ------------------------------- |
| 语法合法性校验 | 编译前  | Error   | 脚本存在语法错误，请检查 C# 规范              |
| 安全沙箱校验  | 编译前  | Error   | 包含非法调用（如 Process.Start 或 IO 操作） |
| 执行超时限制  | 运行中  | Warning | 脚本执行超时，已自动强制终止                  |


### 7. 跨模块联动


| 模块             | 方向   | 说明                         |
| -------------- | ---- | -------------------------- |
| M02UnityBridge | 主动通知 | 通过 WebSocket 链路发送执行请求和接收结果 |
| M01编排服务        | 被动接收 | 接收编排任务下发的自动化测试或补丁脚本        |


### 8. 技术实现要点

- **核心类**：`ScriptRunner` (引擎封装), `SandboxGuard` (指令过滤), `ExecutionManager` (状态追踪)。
- **执行流**：代码字符串 -> SandboxGuard 静态分析 -> Roslyn 编译 -> 内存程序集执行 -> 结果序列化返回。
- **性能约束**：默认执行超时 10s，最大支持 30s；单次执行内存增量不得超过 100MB；禁止频繁 GC 操作。

### 9. 验收标准

- [前置条件] -> Unity 项目处于打开状态 -> [操作] -> 发送 `return 1+1;` 代码 -> [期望结果] -> 返回 result 为 "2"。
- [前置条件] -> 编写无限循环代码 -> [操作] -> 发送执行请求 -> [期望结果] -> 10s 后触发 timeout 状态并释放资源。
- [前置条件] -> 输入包含 `System.IO.File.Delete` -> [操作] -> 发送执行请求 -> [期望结果] -> 触发沙箱拦截，报错拒绝执行。
- [前置条件] -> 定义局部变量并返回 -> [操作] -> 发送执行请求 -> [期望结果] -> 成功获取变量值的字符串表示。

### 10. 边界规范

- **空数据**：输入空代码字符串时返回 400 错误。
- **单元素**：执行简单的常量表达式应能正确识别并返回。
- **上下限临界值**：测试 50000 字符上限的长脚本执行稳定性。
- **异常数据恢复**：若脚本导致 Unity 崩溃，模块应在下次重启后清理残留的 executionId。

### 11. 周边可选功能

- P1: 支持预加载常用命名空间（UnityEngine, UnityEditor），减少脚本体积。
- P2: 支持引用自定义程序集。注意：需预留对 .dll 文件路径的配置接口。
