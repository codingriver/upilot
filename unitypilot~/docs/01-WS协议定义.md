# 01-WS协议定义

## 1. 协议目标
定义 `UnityPilot MCP` 在 Python Orchestrator 与 Unity Editor Bridge 间的 WebSocket 通信协议，确保 Windows/macOS 行为一致。

## 2. 连接模型
- 角色：
  - 服务端：Python Orchestrator（监听 `127.0.0.1:8765`）
  - 客户端：Unity Editor Bridge（启动后自动连接）
- 连接范围：仅本机回环地址。
- 会话：每个 Unity Editor 进程维护 1 个 `sessionId`。

## 3. 消息总结构
所有消息统一 JSON 对象：

```json
{
  "id": "string",
  "type": "command|result|event|error|heartbeat|hello",
  "name": "string",
  "payload": {},
  "timestamp": 0,
  "sessionId": "string",
  "protocolVersion": "1.0"
}
```

字段约束：
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 命令-结果配对 ID；事件可使用事件 ID |
| type | string | 是 | 消息类别 |
| name | string | 是 | 命令名/事件名/错误名 |
| payload | object | 是 | 业务数据，允许空对象 `{}` |
| timestamp | long | 是 | Unix 毫秒时间戳 |
| sessionId | string | 是 | 会话 ID |
| protocolVersion | string | 是 | 协议版本，首版固定 `1.0` |

## 4. 生命周期消息
### 4.1 握手
1. Unity -> Orchestrator 发送 `hello`：

```json
{
  "id": "h-001",
  "type": "hello",
  "name": "session.hello",
  "payload": {
    "unityVersion": "2022.3",
    "projectPath": "<absolute-path>",
    "platform": "windows|macos"
  },
  "timestamp": 1710000000000,
  "sessionId": "<uuid>",
  "protocolVersion": "1.0"
}
```

2. Orchestrator -> Unity 回 `result`：

```json
{
  "id": "h-001",
  "type": "result",
  "name": "session.hello",
  "payload": {
    "accepted": true,
    "heartbeatIntervalMs": 2000
  },
  "timestamp": 1710000000100,
  "sessionId": "<uuid>",
  "protocolVersion": "1.0"

```

### 4.2 心跳
- 双方每 `2000ms` 发送一次 `heartbeat`。
- 连续 `3` 次心跳超时判定断线。

```json
{
  "id": "hb-100",
  "type": "heartbeat",
  "name": "session.heartbeat",
  "payload": {},
  "timestamp": 1710000002000,
  "sessionId": "<uuid>",
  "protocolVersion": "1.0"
}
```

### 5.5 M27 扩展命令（摘要）
以下命令沿用同一 `command` / `result` 信封；`payload` 字段与 MCP 层一致，仅列名称与要点：

| `name` | 说明 |
| --- | --- |
| `editor.window.close` | `windowTitle`、`matchMode`（exact \| contains） |
| `editor.window.setRect` | `windowTitle`、`matchMode`、`x`、`y`、`width`、`height` |
| `uitoolkit.scrollbar.drag` | `targetWindow`、`scrollViewElementName`、`scrollbarAxis`、`normalizedThumbPosition`、`dragSteps` |
| `uitoolkit.scroll` | 可选 `scrollViewNamePath`（嵌套 ScrollView：`a|b`） |
| `uitoolkit.scrollbar.drag` | 可选 `scrollViewNamePath`；否则 `scrollViewElementName` |
| `uitoolkit.event` | `wheel`：`wheelDeltaX`/`wheelDeltaY`；`capturePointer`/`releasePointer`：`mouseButton` 为 `pointerId` |

### 5.6 M25 RShell（UDP 在 Unity Editor Bridge 内）

| `name` | 说明 |
| --- | --- |
| `rshell.connect` | `host`、`port`、`timeoutMs`、`maxRetries` |
| `rshell.disconnect` | `connectionId` |
| `rshell.status` | 可选 `connectionId`；空则列出全部 |
| `rshell.execute` | `connectionId`、`expression` |
| `rshell.scene_list` | `connectionId` |
| `rshell.scene_info` | `connectionId`、`path` |
| `rshell.get_value` / `rshell.set_value` / `rshell.call_method` | 见 `docs/M25-远程运行时调试模块-需求文档.md` |

Python 编排层**不**实现 UDP；仅转发上述命令。

## 8. 错误消息

```json
{
  "id": "cmd-xxx-001",
  "type": "error",
  "name": "command.error",
  "payload": {
    "code": "UNITY_NOT_CONNECTED",
    "message": "Unity 未连接",
    "detail": {
      "command": "compile.request"
    }
  },
  "timestamp": 1710000040000,
  "sessionId": "<uuid>",
  "protocolVersion": "1.0"
}
```

## 9. 错误码规范
| 错误码 | 语义 | 处理策略 |
| --- | --- | --- |
| UNITY_NOT_CONNECTED | Unity 未连接 | 终止命令并提示重连 |
| COMMAND_TIMEOUT | 命令执行超时 | 标记失败，可重试 |
| INVALID_PAYLOAD | 参数不合法 | 直接拒绝，不重试 |
| COMMAND_NOT_FOUND | 未注册命令 | 直接拒绝 |
| INTERNAL_ERROR | 桥接内部异常 | 返回错误并记录日志 |
| EDITOR_BUSY | Editor 忙碌（如编译中） | 延迟重试或排队 |

## 10. 超时与重试
1. 默认命令超时：`30000ms`。
2. `compile.request` 超时：`120000ms`。
3. 重试策略：
   - `UNITY_NOT_CONNECTED`：先重连，不自动重放命令。
   - `EDITOR_BUSY`：最多重试 3 次，间隔 1000ms。
   - 其他错误：不自动重试。

## 11. 兼容与扩展
- `protocolVersion` 不一致时，返回 `INVALID_PAYLOAD` 并拒绝业务命令。
- 新命令必须保持向后兼容：已有字段不可删除，仅允许新增可选字段。
- 未来扩展字段统一放入 `payload`，不得新增顶层破坏字段。
