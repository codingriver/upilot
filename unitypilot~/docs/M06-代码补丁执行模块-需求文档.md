# M06-代码补丁执行模块-需求文档

## 1. 模块职责

- 负责：
  - 根据修复策略生成的补丁对本地代码文件执行写入。
  - 提供写入前后差异摘要与写入结果。
  - 保证单文件写入原子性（成功全量替换，失败不落盘）。
- 不负责：
  - 不负责补丁生成策略。
  - 不负责 Git 提交。
  - 不负责跨仓库文件分发。
- 输入/输出：
  - 输入：文件路径、旧片段、新片段或整文件内容。
  - 输出：写入成功/失败、差异摘要。

## 2. 数据模型


| 字段名          | 类型     | 语义      | 合法范围                                  | 默认值             |
| ------------ | ------ | ------- | ------------------------------------- | --------------- |
| patchId      | string | 补丁任务 ID | 非空 UUID                               | ""              |
| filePath     | string | 待修改文件路径 | 非空相对或绝对路径                             | ""              |
| patchType    | string | 补丁类型    | `replace_block/replace_all/overwrite` | "replace_block" |
| oldContent   | string | 旧内容片段   | 任意字符串                                 | ""              |
| newContent   | string | 新内容片段   | 任意字符串                                 | ""              |
| writeStatus  | string | 写入状态    | `pending/success/failed/skipped`      | "pending"       |
| errorMessage | string | 写入失败原因  | 任意字符串                                 | ""              |


## 3. CRUD 操作


| 操作       | 入口            | 禁用条件          | Command类名                 | Undo语义            |
| -------- | ------------- | ------------- | ------------------------- | ----------------- |
| 创建补丁任务   | `patch.apply` | `filePath` 为空 | CreatePatchTaskCmd        | 不进 CommandHistory |
| 更新补丁状态   | 写入完成后         | 任务不存在         | UpdatePatchStatusCmd      | 不进 CommandHistory |
| 删除临时写入文件 | 写入成功后         | 临时文件不存在       | RemoveTempPatchFileCmd    | 不进 CommandHistory |
| 创建差异摘要记录 | 写入成功后         | 无差异           | CreatePatchDiffSummaryCmd | 不进 CommandHistory |
| 删除失败补丁任务 | 重试前清理         | 任务不存在         | RemoveFailedPatchTaskCmd  | 不进 CommandHistory |


## 4. 交互规格

- 触发事件：
  - 自动修复循环调用 `patch.apply`。
- 状态变化：
  - `pending -> success/failed/skipped`。
- 数据提交时机：
  - 落盘完成后立即提交结果。
- 取消/回退：
  - 写入失败时回退到旧文件内容，不保留半写入状态。

## 5. 视觉规格

不涉及。

## 6. 校验规则


| 规则                              | 检查时机 | 级别    | 提示文案                    |
| ------------------------------- | ---- | ----- | ----------------------- |
| 文件必须存在                          | 写入前  | Error | `文件不存在：{filePath}`      |
| `replace_block` 必须匹配唯一旧片段       | 写入前  | Error | `旧片段未唯一匹配：{filePath}`   |
| `newContent` 不得为空（overwrite 除外） | 写入前  | Error | `新内容为空，无法写入：{filePath}` |
| 写入后必须可读回校验                      | 写入后  | Error | `写入校验失败：{filePath}`     |


## 7. 跨模块联动


| 模块            | 方向   | 说明            |
| ------------- | ---- | ------------- |
| M04 自动修复循环模块  | 被动接收 | 接收每轮补丁并返回写入结果 |
| M01 编排服务模块    | 主动通知 | 回传补丁执行结果和错误信息 |
| M03 编译与错误收集模块 | 被动接收 | 补丁完成后触发下一轮编译  |


## 8. 技术实现要点

- 关键类与职责：
  - `PatchApplyService`：补丁执行与回滚。
  - `PatchMatcher`：定位替换片段。
  - `PatchAuditLogger`：记录差异摘要。
- 核心流程：
  - `validate -> backup temp -> apply -> verify -> commit/rollback`。
- 性能约束：
  - 单文件 1MB 内写入耗时 `< 100ms`。
  - 单轮补丁任务上限 20 个文件。
  - 禁止使用非原子写入覆盖（必须通过临时文件替换）。

## 9. 验收标准

1. [目标文件存在] -> [执行 `replace_block`] -> [返回 `success` 且文件内容变更生效]。
2. [旧片段匹配 2 处] -> [执行 `replace_block`] -> [返回唯一匹配错误，不写入文件]。
3. [模拟写入异常] -> [执行补丁] -> [文件内容保持原样]。
4. [补丁执行成功] -> [读取审计日志] -> [存在差异摘要记录]。

## 10. 边界规范

- 空数据：`oldContent` 为空且类型为 `replace_block` 时直接拒绝。
- 单元素：只修改 1 个文件时仍输出数组格式执行结果。
- 上下限临界值：0 字节文件与 1MB 文件均可执行 `overwrite`。
- 异常数据恢复：写入中断后自动回滚备份文件并返回失败。

## 11. 周边可选功能

- P1：按语言感知补丁（AST 级），预留 `patchLanguage` 字段。
- P2：补丁冲突自动重试，预留 `retryPolicy` 配置。
