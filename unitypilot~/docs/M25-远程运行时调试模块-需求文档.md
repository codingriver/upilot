# M25-远程运行时调试模块-需求文档

> 基于 [RShell](https://github.com/CharlesFeng207/RShell) 协议的远程设备运行时调试桥接。

---

## 0. 核心架构（实现准则）

| 层级 | 职责 |
|------|------|
| **设备 Player** | 集成 RShell 运行时（`RShell.Shell.Listen(port)`），UDP 收包、执行表达式、按 [RShell `UdpHost`](https://github.com/CharlesFeng207/RShell/blob/main/Runtime/UdpHost.cs) 格式 JSON 分片回包。 |
| **Unity Editor Bridge（C#）** | **唯一实现** UDP 客户端、握手 `hi`/`welcome`、JSON 分片重组、心跳、连接表；注册 WS 命令 `rshell.*`，与设备通信。 |
| **MCP 编排（Python）** | **仅转发**：`dispatcher.call("rshell.xxx", payload)`，**不实现** UDP、不解析 RShell 分片。 |
| **MCP 工具** | 暴露 `unity_rshell_*`，参数映射到上述 WS 命令。 |

**原则**：网络与协议与 Unity 版本、IL2CPP 行为在 **同一仓库的 C#** 维护；Python 保持薄封装，与现有 `compile.*`、`uitoolkit.*` 一致。

**前置**：远程调试需 **Unity Editor 已连接 Bridge**；无 Editor 时无法通过本路径使用 RShell（若需无头 CI 直连设备，属周边可选，非本模块默认）。

---

## 1. 模块职责

- 负责：
  - 在 **Editor Bridge** 内通过 UDP 与真机 / Standalone Build 上的 RShell 运行时通信（协议与 [RShell `UdpHost`](https://github.com/CharlesFeng207/RShell/blob/main/Runtime/UdpHost.cs) 对齐：`hi` 握手、表达式为 **UTF-8 明文**、响应为 **JsonUtility 可解析的 Message 分片**）。
  - 将 MCP 工具调用转为 WS 命令 → Bridge 再转为 RShell 表达式或握手包。
  - 管理连接生命周期：握手、心跳、超时、分片重组（均在 **C#** 完成）。
  - 提供场景列表、对象信息、属性读写、方法调用等高层命令（表达式构造在 C#）。
- 不负责：
  - 在设备上安装 RShell（由用户集成到 Build）。
  - 替代 M19/M20 在 Editor 内的能力。
  - 远程设备本地文件系统、设备间中转。
- 输入：MCP / WS 的 `host`、`port`、`connectionId`、表达式等。
- 输出：JSON 结果字符串或标准错误（经 WS 返回 Python）。

## 2. 数据模型

（字段语义与旧版一致；`connectionId` 由 Bridge 生成并持有 UDP 会话。）

### 2.1 连接配置

| 字段名 | 类型 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- |
| connectionId | string | 连接唯一标识 | Bridge 生成 | — |
| host | string | 目标 IPv4 | 合法 IPv4 | `127.0.0.1` |
| port | int | RShell 监听端口 | [1024, 65535] | 9999 |
| status | string | disconnected / connecting / connected / error | 枚举 | disconnected |
| lastHeartbeatAt | long | 最近心跳成功 Unix ms | ≥ 0 | 0 |
| timeoutMs | int | 单次请求超时 | [1000, 60000] | 10000 |
| maxRetries | int | 握手等重试次数 | [0, 20] | 3 |

### 2.2 执行请求

| 字段名 | 类型 | 语义 |
| --- | --- | --- |
| requestId | string | 请求 ID |
| connectionId | string | 目标连接 |
| expression | string | RShell 表达式，1–10000 字符 |
| status | string | pending / completed / failed / timeout |
| result | string | 远程返回文本 |
| roundTripMs | long | 往返耗时 ms |

### 2.3 分片（RShell 与 Bridge 共用内部模型）

与 RShell `UdpHost.Message` 一致：`MsgId`、`FragIndex`、`FragCount`、`Content`、`CheckMsg`（JSON UTF-8 传输）。

## 4. 交互规格

（握手、执行、分片、心跳逻辑与 RShell 开源实现一致；**实现位置：Editor Bridge C#**。）

### 4.1 连接

1. 校验 IPv4、端口、并发连接数（≤10）。
2. `UdpClient` 连接 `host:port`。
3. 发送 UTF-8 字节 `"hi"`。
4. 接收 JSON 分片直至拼出含 `welcome` 的完整内容（不区分大小写）。
5. 超时 / 重试按 `timeoutMs`、`maxRetries`。
6. 成功后注册 `connectionId`，可启动心跳定时器（每 5s 发 `hi`，连续 3 次无 `welcome` 则 `error`）。

### 4.2 执行

1. 校验 `connectionId`、`status==connected`。
2. 将 `expression` 以 **UTF-8 明文** 发送（非 JSON）。
3. 接收 JSON 分片，按 `MsgId` 重组 `Content`。
4. 超时返回 `timeout` / `failed`。

### 4.3 高层表达式映射（C# 内构造）

| 命令 | 生成的表达式 |
| --- | --- |
| `rshell.scene_list` | `Scene.List()` |
| `rshell.scene_info` | `Scene.Info("path")`（path 转义） |
| `rshell.get_value` | 直接使用 `expression` |
| `rshell.set_value` | `{expression}={value}` |
| `rshell.call_method` | `{expression}({args})` |

## 8. 技术实现要点

### 8.1 关键类（C#，`unitypilot-editor/Editor/`）

| 类名 | 职责 |
| --- | --- |
| `UnityPilotRShellService` | 注册 `rshell.*`，会话表，`connectionId` 生成 |
| `RShellWireMessage` | 与 RShell 对齐的 JSON 分片模型 |
| `RShellFragmentAssembler` | 按 `MsgId` 重组 `Content` |
| （内部） | `UdpClient`、心跳定时器、锁（每连接串行化 UDP） |

### 8.2 数据流

```
MCP 客户端
  -> Python: unity_rshell_connect / execute / ...
    -> WebSocket: command rshell.* + JSON payload
      -> UnityPilotBridge -> UnityPilotRShellService
        -> UdpClient Send/Receive（与设备 RShell UdpHost 对话）
    <- result JSON（含 result / error / status）
```

### 8.3 Python 实现范围

仅：`src/unitypilot_mcp/tool_facade.py`（`rshell_*` 方法）、`mcp_stdio_server.py`（工具注册）、`dispatcher.py`（`rshell.*` 超时）。**禁止**在 Python 中实现 UDP 或分片重组。

### 8.4 性能与安全

- 单连接内 UDP 操作加锁；分片总大小上限 10MB；并发连接 ≤10。
- 域重载 / Editor 退出时关闭所有 `UdpClient`，清空会话。

### 8.5 禁止项

- 禁止在 Python 侧求值远程表达式。
- 禁止 Python 绑定 `0.0.0.0` 监听；仅 C# 客户端 `Connect/Send/Receive`。
