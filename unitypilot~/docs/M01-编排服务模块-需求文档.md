# M01-编排服务模块-需求文档

## 1. 模块职责

- 负责：
  - 作为 Python WebSocket 服务端，维护 Unity Editor 连接会话。
  - 接收 Agent 调用并路由为统一命令协议。
  - 维护命令生命周期（受理、超时、完成、失败）。
  - 对外提供 `open_editor`、`compile`、`compile_status`、`compile_errors`、`playmode_set`、`mouse_event` 能力入口。
- 不负责：
  - 不负责 Unity Editor 内部 API 调用细节。
  - 不负责代码修复算法本体。
  - 不负责 Unity 业务数据序列化。
- 输入/输出：
  - 输入：Agent 工具调用参数、Unity Bridge 事件消息。
  - 输出：标准化结果对象、错误对象、状态事件。

## 2. 数据模型


| 字段名                 | 类型     | 语义               | 合法范围             | 默认值   |
| ------------------- | ------ | ---------------- | ---------------- | ----- |
| sessionId           | string | 当前 Unity 连接会话 ID | 非空 UUID 字符串      | ""    |
| unityConnected      | bool   | Unity 是否已连接      | `true/false`     | false |
| currentProjectPath  | string | 当前工程路径           | 非空绝对路径或空串        | ""    |
| commandTimeoutMs    | int    | 单命令超时时间          | `[1000, 120000]` | 30000 |
| heartbeatIntervalMs | int    | 心跳间隔             | `[1000, 10000]`  | 2000  |
| reconnectWaitMs     | int    | 断线后等待重连时长        | `[1000, 30000]`  | 5000  |
| lastError           | string | 最近一次编排层错误摘要      | 任意字符串            | ""    |


## 3. CRUD 操作


| 操作     | 入口                              | 禁用条件                  | Command类名                | Undo语义            |
| ------ | ------------------------------- | --------------------- | ------------------------ | ----------------- |
| 创建会话   | `session.hello` 握手              | 已存在同 `sessionId` 活跃会话 | CreateOrUpdateSessionCmd | 不进 CommandHistory |
| 更新会话状态 | `heartbeat` / `editor.state` 事件 | 会话不存在                 | UpdateSessionStateCmd    | 不进 CommandHistory |
| 删除会话   | `session.disconnect` / 心跳超时     | 会话已不存在                | RemoveSessionCmd         | 不进 CommandHistory |
| 创建命令记录 | 任意 MCP 工具调用                     | Unity 未连接且命令要求在线执行    | CreateCommandRecordCmd   | 不进 CommandHistory |
| 更新命令结果 | Unity 返回 `result/error`         | 命令记录不存在               | CompleteCommandRecordCmd | 不进 CommandHistory |


## 4. 交互规格

- 触发事件：
  - Agent 调用 MCP 工具后，编排服务生成 `commandId` 并发送到 Unity。
  - Unity 回传 `result` 或 `error`。
- 状态变化：
  - `pending -> sent -> success/failed/timeout`。
- 数据提交时机：
  - 收到 Unity 回包即提交命令最终态。
  - 超时由本地定时器提交 `timeout` 最终态。
- 取消/回退：
  - MVP 不支持主动取消；命令超时后标记失败，不重放旧命令。

## 5. 视觉规格

不涉及。

## 6. 校验规则


| 规则                   | 检查时机                   | 级别      | 提示文案                          |
| -------------------- | ---------------------- | ------- | ----------------------------- |
| Unity 连接必须存在         | MCP 工具执行前              | Error   | `Unity 未连接，命令 {command} 无法执行` |
| 消息必须含 `id/type/name` | 收到任意 WS 消息时            | Error   | `协议字段缺失：{field}`              |
| 命令超时必须终止等待           | `commandTimeoutMs` 到达时 | Error   | `命令超时：{commandId}`            |
| 未知命令名直接拒绝            | 路由前                    | Warning | `未知命令：{name}`                 |

| 模块                  | 方向   | 说明                  |

## 8. 技术实现要点

- 关键类与职责：
  - `WsOrchestratorServer`：WebSocket Server 生命周期管理。
  - `SessionRegistry`：会话注册、断线清理、在线状态。
  - `CommandDispatcher`：命令下发、超时、结果归档。
  - `McpToolFacade`：对外暴露工具函数。
- 核心流程：
  - `tool call -> validate -> create command record -> send ws command -> await result -> return`。
- 性能约束：
  - 单会话下并发命令上限 32。
  - 心跳处理不得阻塞命令线程；单次心跳处理时间 `< 5ms`。
  - 禁止在消息回调中执行磁盘全量扫描。

## 9. 验收标准

1. [Unity 已连接] -> [调用 `compile`] -> [返回 accepted，且包含 `commandId`]。
2. [发送非法协议消息] -> [编排服务解析] -> [返回字段缺失错误，不导致服务崩溃]。
3. [Unity 30 秒无响应] -> [等待超时] -> [命令状态变为 `timeout` 并返回失败结果]。
4. [Unity 断开连接] -> [再次调用 `playmode_set`] -> [返回 `Unity 未连接` 错误]。

## 10. 边界规范

- 空数据：收到空 `payload` 时，若命令必填参数缺失则直接拒绝，返回字段缺失错误。
- 单元素：仅 1 个会话时，所有命令默认路由到该会话。
- 上下限临界值：`commandTimeoutMs=1000` 与 `120000` 均应正确生效。
- 异常数据恢复：解析异常或网络抖动后，会话状态应在下一次心跳中恢复为 `connected=true`。

## 11. 周边可选功能

- P1：命令取消（`cancel_command`），预留 `commandStatus=cancelled` 枚举值与取消事件。
- P2：多 Unity 实例路由，预留 `projectId -> sessionId` 映射接口。
