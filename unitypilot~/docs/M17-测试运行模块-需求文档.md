# M17-测试运行模块-需求文档

## 1. 模块职责
- 负责：
  - 提供 Unity Test Framework (UTF) 测试运行触发入口。
  - 执行 EditMode（编辑模式）与 PlayMode（播放模式）下的自动化测试。
  - 收集测试执行过程中的状态与结果详情。
  - 按模式或过滤器筛选并列出可用测试。
- 不负责：
  - 不负责编写或修改测试脚本。
  - 不负责测试环境（如测试用例所需的外部资源）的准备。
  - 不负责覆盖率报告的生成。
- 输入/输出：
  - 输入：`test.run`, `test.results`, `test.list` 命令。
  - 输出：`test.started` 事件、`test.results` 结构化报告。

## 2. 数据模型
| 字段名 | 类型 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- |
| testMode | string | 测试运行模式 | EditMode/PlayMode | "EditMode" |
| testName | string | 测试用例全名 | 非空字符串 | "" |
| testStatus | string | 测试执行状态 | Passed/Failed/Skipped/Inconclusive | "Inconclusive" |
| duration | float | 测试耗时（秒） | [0, +∞) | 0.0 |
| message | string | 失败详细信息 | 字符串 | "" |
| stackTrace | string | 失败堆栈信息 | 字符串 | "" |

## 3. CRUD 操作
| 操作 | 入口 | 禁用条件 | Command类名 | Undo语义 |
| --- | --- | --- | --- | --- |
| 执行测试任务 | test.run | 已有测试进行中 | RunTestsCmd | 不进 CommandHistory |
| 查询测试结果 | test.results | 无 | GetTestResultsCmd | 不进 CommandHistory |
| 获取测试列表 | test.list | 无 | ListAvailableTestsCmd | 不进 CommandHistory |
| 更新单项结果 | 测试回调事件 | 任务已取消 | UpdateTestItemResultCmd | 不进 CommandHistory |
| 清除历史报告 | 新运行开始前 | 无 | ClearTestHistoryCmd | 不进 CommandHistory |

## 4. 交互规格
- 触发事件：收到 test.run 后通过 Unity API 调用 TestRunnerApi 启动测试。
- 状态变化：idle -> running -> completed。
- 数据提交时机：测试开始时上报任务 ID；每项测试完成时上报实时状态；所有完成后上报汇总报告。
- 取消/回退：MVP 版本不支持中途强行中断测试；支持指定特定的测试类或方法过滤运行。

## 5. 视觉规格
不涉及。

## 6. 校验规则
| 规则 | 检查时机 | 级别 | 提示文案 |
| --- | --- | --- | --- |
| PlayMode 运行环境校验 | 启动 PlayMode 测试前 | Error | 必须先退出当前 PlayMode 或编译中 |
| 过滤器格式校验 | 收到 test.run 时 | Warning | 无效的测试类名或过滤器字符串 |
| 连续触发锁定 | 命令执行期间 | Error | 正在运行测试，请勿重复触发 |
| 测试存在性检查 | 启动特定测试前 | Warning | 未在项目中找到指定的测试用例 |

## 7. 跨模块联动
| 模块 | 方向 | 说明 |
| --- | --- | --- |
| M02 UnityBridge通信模块 | 被动接收 | 接收测试命令并实时透传执行进度 |
| M05 PlayMode与输入模拟模块 | 主动通知 | PlayMode 测试需协调进入/退出播放模式 |
| M01 编排服务模块 | 被动接收 | 返回最终测试通过率作为编排决策依据 |

## 8. 技术实现要点
- 关键类与职责：
  - UnityTestRunner：封装 TestRunnerApi 的调用与监听。
  - TestResultAccumulator：负责结构化并缓存测试运行结果。
  - TestDiscoveryManager：负责扫描并维护项目测试用例清单。
- 核心流程：test.run -> check playmode -> clear cache -> api call -> callback listeners -> package results -> publish。
- 性能约束：
  - 测试列表发现耗时 < 200ms。
  - 结果实时上报延迟 < 50ms。
  - 内存中仅保留最近 3 次运行的详细结果数据。

## 9. 验收标准
1. [空闲状态] -> [调用 test.list] -> [返回 EditMode 和 PlayMode 下所有可见测试列表]。
2. [代码逻辑正确] -> [调用 test.run] -> [所有用例 testStatus="Passed"]。
3. [断言失败用例] -> [测试完成] -> [message 包含预期错误文案，stackTrace 包含代码位置]。
4. [PlayMode 已启动] -> [调用 EditMode 测试] -> [返回 403 环境冲突错误]。

## 10. 边界规范
- 空数据：项目中没有任何测试文件时 test.list 返回 []。
- 单元素：仅指定运行单个方法时，results 返回长度为 1 的数组。
- 上下限临界值：支持一次运行超过 1000 个轻量级单元测试。
- 异常数据恢复：Unity 闪退或测试死锁时，超时逻辑强制设置当前任务为 Inconclusive 并释放锁定。

## 11. 周边可选功能
- P1：增量测试运行，预留 modifiedFilesOnly 过滤器。
- P2：测试资源泄露检测，预留 heapDelta 监控标记。
