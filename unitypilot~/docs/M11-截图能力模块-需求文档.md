# M11-截图能力模块-需求文档

## 1. 模块职责
- 负责：
  - 获取 Unity Game 视图的实时画面截图。
  - 获取 Unity Scene 视图的实时编辑器画面截图。
  - 获取指定 Camera 的画面内容（Base64 编码）。
  - 支持指定分辨率、格式（PNG/JPG）和质量。
- 不负责：
  - 不负责视频录制与流媒体传输。
  - 不负责图像的后期处理（如滤镜、修剪）。
  - 不负责截图文件的长期存储。
- 输入/输出：
  - 输入：`screenshot.gameView`, `screenshot.sceneView`, `screenshot.camera` 命令。
  - 输出：包含 Base64 字符串的 `imageData` 及元数据（宽高、格式）。

## 2. 数据模型
| 字段名 | 类型 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- |
| imageData | string | 截图的 Base64 编码数据 | 非空 Base64 | "" |
| width | int | 输出图像宽度（像素） | `[1, 4096]` | 1280 |
| height | int | 输出图像高度（像素） | `[1, 4096]` | 720 |
| cameraName | string | 目标相机名称 | 场景中存在的相机名 | "" |
| format | string | 图像编码格式 | `png/jpg` | "png" |
| quality | int | JPG 压缩质量 | `[1, 100]` | 75 |

## 3. CRUD 操作
| 操作 | 入口 | 禁用条件 | Command类名 | Undo语义 |
| --- | --- | --- | --- | --- |
| 捕获 Game 视图 | `screenshot.gameView` | 编辑器关闭 | CaptureGameViewCmd | 不进 CommandHistory |
| 捕获 Scene 视图 | `screenshot.sceneView` | 无 Scene 窗口 | CaptureSceneViewCmd | 不进 CommandHistory |
| 捕获指定相机 | `screenshot.camera` | 找不到相机 | CaptureCameraCmd | 不进 CommandHistory |
| 刷新截图缓存 | 每一帧开始时 | 无 | ClearScreenshotCacheCmd | 不进 CommandHistory |
| 设置截图参数 | 指令下发时 | 参数越界 | SetScreenshotConfigCmd | 不进 CommandHistory |

## 4. 交互规格
- 触发事件：
  - 收到截图指令，调用 `ScreenCapture` 或 `RenderTexture` 逻辑。
- 状态变化：
  - `idle -> capturing -> encoding -> result` |
- 数据提交时机：
  - 完成编码后，将 Base64 数据通过 WebSocket 发送。
- 取消/回退：
  - 截图过程不可中断；超时将返回 `TimeoutError`。

## 5. 视觉规格
不涉及。

## 6. 校验规则
| 规则 | 检查时机 | 级别 | 提示文案 |
| --- | --- | --- | --- |
| 相机必须激活 | 指定相机截图前 | Warning | `相机 {cameraName} 处于禁用状态，可能导致黑屏` |
| 分辨率超限 | 指令校验时 | Error | `宽度或高度超过系统限制 (4096px)` |
| 格式不支持 | 指令校验时 | Error | `格式 {format} 不受支持` |
| Game 窗口丢失 | Game 视图截图前 | Error | `未发现有效的 Game 视图窗口` |

## 7. 跨模块联动
| 模块 | 方向 | 说明 |
| --- | --- | --- |
| M02 UnityBridge通信模块 | 被动接收 | 接收截图指令并回传大容量数据包 |
| M01 编排服务模块 | 被动接收 | 为多模态 AI 提供视觉输入数据 |
| M05 PlayMode 模块 | 主动通知 | 在 PlayMode 切换前后自动触发状态快照 |

## 8. 技术实现要点
- 关键类与职责：
  - `ScreenShotService`：统一截图调度器，管理渲染请求。
  - `RenderTextureProxy`：处理离屏渲染并读取纹理像素。
  - `Base64Encoder`：将字节流高效转换为 Base64 字符串。
- 核心流程：
  - `screenshot.camera -> set target texture -> render -> read pixels -> encode to png -> base64 -> reply`。
- 性能约束：
  - 1080P PNG 截图从请求到回包全过程 `< 200ms`。
  - 编码过程在后台线程执行，避免卡顿编辑器主线程。
  - 限制短时间内重复截图的频率。

## 9. 验收标准
1. [空闲状态] -> [调用 `screenshot.gameView`] -> [返回合法的 Base64 数据串]。
2. [指定无效相机名] -> [调用 `screenshot.camera`] -> [返回错误：找不到相机]。
3. [设置格式为 jpg] -> [调用截图] -> [返回数据包体积明显小于 png]。
4. [频繁请求截图] -> [系统检测频率] -> [第二条及后续请求返回成功或限流提醒]。

## 10. 边界规范
- 空数据：渲染空场景时，返回正常的背景色截图（非 null）。
- 单元素：仅一个相机的场景，`screenshot.camera` 默认捕获该相机。
- 上下限临界值：在 1x1 分辨率下截图，应能正常工作。
- 异常数据恢复：GPU 显存不足时，自动回退到较低分辨率并报错。

## 11. 周边可选功能
- P1：支持裁剪矩形区域截图，预留 `targetRect` 字段。
- P2：支持深度贴图导出，预留 `exportDepthMap` 接口。
