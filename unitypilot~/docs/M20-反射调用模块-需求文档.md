# M20-反射调用模块-需求文档

### 1. 模块职责
- 负责通过反射机制搜索 Unity 项目及编辑器程序集中的类和方法。
- 负责动态调用指定类的方法，并支持传递复杂参数。
- 负责将方法调用结果转换为字符串返回。
- 不负责执行具有破坏性的反射操作（如修改只读私有字段）。
- 不负责跨程序集注入新的代码逻辑。
- 不负责处理不支持序列化的非基元类型返回值。

### 2. 数据模型
| 字段名 | 类型 | 语义 | 合法范围 | 默认值 |
| :--- | :--- | :--- | :--- | :--- |
| typeName | string | 全限定类名 | 符合 C# 命名空间规范 | 无 |
| methodName | string | 方法名 | 长度 1-100 字符 | 无 |
| parameters | list | 参数列表 | JSON 对象数组 | 空列表 |
| isStatic | bool | 是否为静态方法 | true/false | true |
| targetInstancePath | string | 实例对象层级路径 | 场景中 Hierarchy 路径 | 空 |
| returnValue | string | 执行结果序列化 | 任意字符串 | 空 |
| error | string | 调用失败原因 | 异常堆栈或错误描述 | 空 |

### 3. CRUD 操作
| 操作 | 入口 | 禁用条件 | Command 类名 | Undo 语义 |
| :--- | :--- | :--- | :--- | :--- |
| 查找方法 | reflection.find | 类名不存在 | FindMethodCmd | 不进 CommandHistory |
| 调用方法 | reflection.call | 目标对象未找到 | CallMethodCmd | 不进 CommandHistory |
| 注册白名单 | reflection.whitelist | 无 | UpdateWhitelistCmd | 不进 CommandHistory |

### 4. 交互规格
- **触发事件**：客户端发送 `reflection.find` 后返回方法列表。
- **状态变化**：同步执行调用后立即返回结果。
- **数据提交时机**：反射调用完成并捕获输出后提交返回包。
- **取消回退**：反射调用产生的副作用无法撤回。

### 5. 视觉规格
不涉及。

### 6. 校验规则
| 规则 | 检查时机 | 级别 | 提示文案 |
| :--- | :--- | :--- | :--- |
| 白名单校验 | 调用前 | Error | 该程序集/方法不在安全调用白名单内 |
| 参数匹配校验 | 调用前 | Error | 传入参数个数或类型与目标方法不符 |
| 实例存在性校验 | 调用前 | Error | 指定的 Hierarchy 路径下未找到组件实例 |

### 7. 跨模块联动
| 模块 | 方向 | 说明 |
| :--- | :--- | :--- |
| M02UnityBridge | 被动接收 | 监听控制端发起的反射查询与调用指令 |
| M01编排服务 | 主动通知 | 反射执行状态及异常捕获上报 |

### 8. 技术实现要点
- **核心类**：`ReflectionSearcher` (搜索逻辑), `MethodInvoker` (调用网关), `TypeConverter` (参数转换)。
- **执行流**：输入 type/method -> ReflectionSearcher 定位 MethodInfo -> TypeConverter 处理参数 -> MethodInvoker 触发执行 -> 结果上报。
- **性能约束**：反射查找需具备内部缓存机制（Cache Hit）；单次调用耗时需监控，超出 2s 报警；避免在大循环中使用反射调用。

### 9. 验收标准
- [前置条件] -> Unity 场景中存在名为 "Cube" 的物体 -> [操作] -> 反射调用 Cube.SetActive(false) -> [期望结果] -> 物体被隐藏。
- [前置条件] -> 输入非法类名 -> [操作] -> 发送 `reflection.find` -> [期望结果] -> 返回 404，提示类未找到。
- [前置条件] -> 调用私有内部 API -> [操作] -> 无授权调用 -> [期望结果] -> 触发权限拦截。
- [前置条件] -> 调用成功 -> [操作] -> 获取返回值 -> [期望结果] -> 正确返回方法结果的字符串。

### 10. 边界规范
- **空数据**：空方法名导致查询失败。
- **单元素**：调用无参方法时 `parameters` 传空数组应能成功。
- **上下限临界值**：测试具有 20 个参数的方法调用稳定性。
- **异常数据恢复**：调用导致异常（Exception）时需完整捕获并返回堆栈信息，避免阻塞进程。

### 11. 周边可选功能
- P1: 支持对字段/属性的动态读取。
- P2: 支持泛型方法的调用注入。注意：需预留对 T 类型的解析逻辑。
