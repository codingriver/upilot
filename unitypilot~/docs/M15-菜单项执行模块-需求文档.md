# M15-菜单项执行模块-需求文档

## 1. 模块职责
- 负责：
  - 执行 Unity 编辑器中任何有效的菜单项路径。
  - 获取并列出当前环境中所有可调用的菜单项及其状态。
  - 模拟对特定编辑器功能的调用。
- 不负责：
  - 不负责菜单项内部的具体逻辑实现。
  - 不负责菜单项路径的自动生成（需由前端提供或扫描得出）。
  - 不负责带对话框弹出的菜单项交互（会自动跳过或导致阻塞）。
- 输入/输出：
  - 输入：`menu.execute`、`menu.list` 命令。
  - 输出：`menu.executed` 事件、菜单项列表对象。

## 2. 数据模型
| 字段名 | 类型 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- |
| menuPath | string | 菜单完整路径 | 包含 `/` 的路径字符串 | "" |
| isAvailable | bool | 该菜单项当前是否可用 | `true/false` | false |
| shortcut | string | 菜单项快捷键 | 系统快捷键字符串 | "" |
| executeResult | bool | 最近一次执行结果 | `true/false` | false |

## 3. CRUD 操作
| 操作 | 入口 | 禁用条件 | Command类名 | Undo语义 |
| --- | --- | --- | --- | --- |
| 执行菜单项 | `menu.execute` | `isAvailable=false` | ExecuteMenuItemCmd | 不进 CommandHistory |
| 查询可用菜单 | `menu.list` | 无 | ListMenuItemsCmd | 不进 CommandHistory |
| 刷新菜单状态 | 菜单事件变更 | 无 | RefreshMenuStatusCmd | 不进 CommandHistory |

## 4. 交互规格
- 触发事件：接收到 `menu.execute` 后调用 `EditorApplication.ExecuteMenuItem`。
- 状态变化：`Ready -> Executing -> Finished`。
- 数据提交时机：
  - 执行完成后立即返回布尔值。
  - 列表查询时上报完整的 `menuPath` 数组。
- 取消/回退：部分系统级菜单操作无法撤销，建议由具体业务模块管理 Undo。

## 5. 视觉规格
不涉及。

## 6. 校验规则
| 规则 | 检查时机 | 级别 | 提示文案 |
| --- | --- | --- | --- |
| 菜单路径必须包含 `/` | 收到执行请求时 | Error | `菜单路径非法，应如 File/Save` |
| 路径对应的菜单项不存在 | 执行 `menu.execute` 前 | Error | `未找到菜单项：{menuPath}` |
| 菜单项当前不可选（灰掉） | 执行 `menu.execute` 前 | Warning | `菜单项当前不可用` |
| 路径包含系统敏感操作 | 执行前校验 | Warning | `慎重：正在执行敏感操作 {menuPath}` |

## 7. 跨模块联动
| 模块 | 方向 | 说明 |
| --- | --- | --- |
| M02 UnityBridge通信模块 | 被动接收 | 接收并返回菜单执行状态 |
| M01 编排服务模块 | 主动通知 | 菜单执行完成触发编排回调 |
| M04 自动修复循环模块 | 被动接收 | 调用 `menu.execute("Asset/Reimport")` 等辅助修复 |

## 8. 技术实现要点
- 关键类与职责：
  - `MenuService`：封装 `EditorApplication.ExecuteMenuItem` 并处理异常。
  - `MenuExplorer`：反射并扫描所有注册在 `MenuItem` 特性上的方法。
  - `MenuStateMonitor`：检测菜单项是否处于 `Validate` 通过状态。
- 核心流程：`menu.execute -> Validate(path) -> ExecuteMenuItem(path) -> Return status`。
- 性能约束：
  - 列出全量菜单项耗时 `< 200ms`。
  - 单个菜单项执行调度延迟 `< 10ms`（不含业务逻辑耗时）。
  - 禁止在主线程频繁轮询菜单状态。

## 9. 验收标准
1. [提供路径 "File/Save"] -> [调用 `menu.execute`] -> [Unity 项目执行保存操作]。
2. [调用 `menu.list`] -> [返回包含 "Edit/Copy"、"GameObject/Create Empty" 等路径]。
3. [提供无效路径 "Tools/NonExist"] -> [调用 `menu.execute`] -> [返回 false 并提示路径不存在]。
4. [选中场景对象执行 "GameObject/Delete"] -> [对象被成功删除]。

## 10. 边界规范
- 空数据：空路径直接返回 `Invalid path`。
- 单元素：支持对自定义工具菜单项的单次调用。
- 上下限临界值：处理深达 10 级的多层级菜单路径无性能抖动。
- 异常数据恢复：菜单执行引起 Unity 报错后，模块需记录日志并重置状态。

## 11. 周边可选功能
- P1：支持带参数的静态方法调用，预留 `method.invoke` 接口。
- P2：菜单项常用度统计，预留 `usageCount` 字段。
