# 05-错误码与排障手册

## 1. 目标
统一 `UnityPilot MCP` 的错误码语义、排查步骤和恢复动作，降低联调与线上排障成本。

## 2. 错误分级
| 级别 | 定义 | 处理时限 |
| --- | --- | --- |
| P0 | 阻塞主流程（无法连接、无法编译、无法执行命令） | 立即处理 |
| P1 | 功能受限（可绕过但影响效率） | 当天处理 |
| P2 | 体验问题（重试可恢复） | 迭代内处理 |

## 3. 标准错误对象

```json
{
  "ok": false,
  "error": {
    "code": "COMMAND_TIMEOUT",
    "message": "命令超时",
    "detail": {
      "command": "compile.request",
      "sessionId": "<uuid>",
      "requestId": "<uuid>"
    }
  },
  "requestId": "<uuid>",
  "timestamp": 1710000000000
}
```

## 4. 错误码总表
| 错误码 | 级别 | 典型场景 | 首选处理 |
| --- | --- | --- | --- |
| UNITY_NOT_CONNECTED | P0 | Unity 未连接或断线 | 先重连会话，再重试命令 |
| COMMAND_TIMEOUT | P0 | Unity 未在超时内回包 | 记录 requestId，检查 Unity 主线程阻塞 |
| INVALID_PAYLOAD | P1 | 参数缺失/类型错误 | 修正请求参数，禁止重试旧请求 |
| COMMAND_NOT_FOUND | P1 | 命令未注册或名称拼写错误 | 校验协议名与路由注册 |
| EDITOR_BUSY | P1 | 编译中再次触发编译/PlayMode | 等待状态结束后重试 |
| INTERNAL_ERROR | P0 | 桥接内部异常 | 抓取堆栈并重启 Bridge |
| WINDOW_NOT_AVAILABLE | P1 | 鼠标目标窗口不可见或不存在 | 先聚焦/打开窗口再执行 |
| PATCH_APPLY_FAILED | P1 | 代码补丁写入失败 | 检查唯一匹配与文件权限 |
| LOOP_MAX_REACHED | P2 | 自动修复达到上限 | 输出残留错误并人工介入 |

## 5. 高频问题排障流程
## 5.1 无法连接 Unity（UNITY_NOT_CONNECTED）
1. 检查 Orchestrator 是否监听 `127.0.0.1:8765`。
2. 检查 Unity Console 是否有 Bridge 启动日志。
3. 检查 `session.hello` 是否成功回包。
4. 若仍失败：重启 Unity，再重启 Orchestrator。

通过标准：30 秒内恢复 `connected=true`。

## 5.2 编译命令超时（COMMAND_TIMEOUT）
1. 记录 `requestId` 与 `compileRequestId`。
2. 检查 Unity 是否处于脚本刷新或导入长任务。
3. 检查是否有死循环日志导致主线程卡住。
4. 临时将 compile 超时从 `30000ms` 提高到 `120000ms` 验证。

通过标准：相同项目下连续 3 次编译均在超时阈值内完成。

## 5.3 参数错误（INVALID_PAYLOAD）
1. 校验 JSON 字段是否完整。
2. 校验枚举字段是否合法（如 `button/action/state`）。
3. 校验坐标和时间参数范围。
4. 修正后重新发送新 `requestId` 请求。

通过标准：同请求参数在两平台都返回 `ok=true`。

## 5.4 命令未注册（COMMAND_NOT_FOUND）
1. 对照 `01-WS协议定义.md` 校验 `name`。
2. 检查 UnityBridge 路由注册表是否含该命令。
3. 检查版本是否一致（`protocolVersion=1.0`）。
4. 重新加载 Bridge 初始化。

通过标准：命令可被路由并返回 result/error，而非 not found。

## 5.5 编辑器忙（EDITOR_BUSY）
1. 查询 `compile_status`，确认是否仍在 compiling。
2. 若是编译引发，等待结束后重放一次命令。
3. 同类命令启用队列化（只保留最后一次请求）。
4. 若长期 busy，检查是否有挂起编译任务。

通过标准：busy 状态结束后 1 次重试即可成功。

## 5.6 鼠标窗口不可用（WINDOW_NOT_AVAILABLE）
1. 校验 `targetWindow` 是否合法。
2. 强制打开并聚焦目标窗口。
3. 校验坐标是否在窗口边界内。
4. 重新发送 `mouse_event`。

通过标准：目标窗口可执行 click 与 drag 并回包成功。

## 5.7 补丁应用失败（PATCH_APPLY_FAILED）
1. 检查文件是否存在、是否只读。
2. 检查 `replace_block` 的旧片段是否唯一。
3. 检查编码是否一致（UTF-8）。
4. 回滚后改用 `overwrite` 或人工修复。

通过标准：补丁应用后可通过下一轮编译。

## 6. 日志定位规范
必须记录以下字段：
| 字段 | 说明 |
| --- | --- |
| timestamp | 毫秒时间戳 |
| requestId | MCP 请求 ID |
| commandId | WS 命令 ID |
| sessionId | Unity 会话 ID |
| commandName | 命令名称 |
| result | success/failed/timeout |
| errorCode | 错误码（成功为空） |

日志建议格式：JSONL（每行一个 JSON 对象）。

## 7. 快速自愈策略
1. 连接断开：自动重连，最多 5 次，间隔 2 秒。
2. 编辑器忙：排队重试，最多 3 次。
3. 超时：标记失败，不自动无限重试。
4. 自动修复循环达到上限：输出剩余错误并终止。

## 8. 双平台差异注意点
| 项目 | Windows | macOS | 处理建议 |
| --- | --- | --- | --- |
| 路径分隔符 | `\` | `/` | 内部统一 POSIX 风格再映射 |
| 窗口焦点行为 | 较稳定 | 偶发切焦失败 | 鼠标前强制聚焦并验证 |
| 启动命令 | `.exe` 或 Hub 命令 | `open -a Unity` | `unity.open_editor` 支持自定义命令 |

## 9. 升级与回归策略
1. 每次协议字段变更必须回归 `04-测试用例与验收脚本清单.md` 全套用例。
2. 新增错误码必须补充本手册和 `02-MCP工具接口定义.md`。
3. Unity 版本升级前必须完成双平台 smoke test。
